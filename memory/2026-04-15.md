## PLATO-OS Dual Render — Product Vision (02:30 UTC)

Casey laid out the complete PLATO-OS vision in a late-night session:

### Text as Ground Truth
- **Text IS the game state. Visuals are projections.** Physical actions are projections. Robot perception is projections.
- Same text state can render as: pixel art, 3D, terminal/MUD, joystick controls, robot servos, stereoscopic camera descriptions
- "The visual doesn't drive anything. It's a read-only projection of the text state."

### Three Modes, Same Engine
```
SIMULATE  →  text state → pixel chart on screen
PLAYTEST  →  text state → pixel chart + joystick input + "what if" scenarios  
DRIVE     →  text state → physical rudder + real GPS + actual water
```
- The agent doesn't know which mode it's in. `course 045` works the same in simulation as on steel.
- Train in simulation, playtest with visuals, deploy on real boats.

### Bidirectional Translation Layer
- Humans see the inside (agent state, text, strategy tiles)
- Agents feel the outside (sensors, relays, physical feedback)
- Each describes what they need in the other's language
- Human says "I need the pump on" → agent translates to "relay_4 HIGH"
- Agent says "blind spot on port stern, recommend ultrasonic" → human wires the sensor
- **The product isn't the hardware. It's the translation layer.**

### Rooms as Cognitive Tools
- **Brainstorm Room** (Reverse Actualization): Start with outcome, work backwards
- **Situation Room** (Pre-Planning): Walk through scenarios, leave script traces
- **War Room** (Live Crisis): Real sensor feeds → text state → pre-planned agent response
- **After-Action Room** (Post-Mortem): Replay timeline, agent learns from real experience

### Repo Types = Physical Objects
- Agent repos → Characters in the world
- Room repos → Spaces on the ship
- Equipment repos → Deck equipment (tools agents use)
- Power Armor repos → Equipment that enhances an agent (seed-mcp = JC1's power armor)
